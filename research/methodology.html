데이터 수집 및 분석 방법

본 사이트는 모두의 창업 공개 아이디어를 기반으로 하며, 수집·집계 절차는 저장소의 Python 모듈(modoo_fetch.py, modoo_analytics.py, modoo_insight.py)에 구현되어 있습니다.

데이터 수집 및 분석 방법 아이디어 목록 Raw data 분석 Insight idea 10

1. 데이터 수집 기간·출처

  • 수집 시점(스냅샷): "최신 데이터 수집"을 실행할 때마다 그 시각이 crawled_at(ISO 날짜시간)으로 기록됩니다. 특정 역사 구간을 고정해 두지 않고, 실행 시점의 공개 목록 전체를 한 번에 가져옵니다.
  • 출처(원천 서비스): 웹 목록 페이지 https://www.modoo.or.kr/idea/list와 동일한 데이터를, 서버가 노출하는 JSON API(https://hera-prod.modoo.or.kr/api/v1/startup-idea)를 통해 수집합니다.
  • 로컬 보관 파일: 수집 결과는 프로젝트 루트의 modoo_all_ideas.json에 저장되고, 연구용 정적보내기 시 research/data/ideas.json에도 복제됩니다.
  • 인사이트 재현용 메타: Insight 화면과 함께 research/data/insight_data_source.json이 생성되며, 전수 행 수·수집 시각·API 엔드포인트 등 재현에 필요한 출처 정보를 담습니다.

2. 데이터 크롤링(수집) 방법

  • 엔드포인트·요청: modoo_fetch.fetch_page가 페이지 번호·페이지 크기(기본 12건)·정렬(createdAt,desc)과 선택적 세부분야 태그(tag)를 쿼리로 붙여 GET 요청합니다. 브라우저와 유사한 User-Agent, Referer(www.modoo.or.kr), Origin 헤더를 사용합니다.
  • 응답 복호화: API는 Base64로 감싼 AES-CBC 암호문과 타임스탬프를 돌려줍니다. 타임스탬프 문자열(왼쪽부터 16자)을 키로 삼아 복호화한 뒤 JSON 배열로 파싱합니다.
  • 전수 수집 전략: modoo_fetch.collect_all_ideas는 사전에 정의한 세부분야(22개) 각각에 대해 첫 응답의 totalPage만큼 페이지를 순회합니다. 페이지 간 짧은 지연(delay, 기본 0.25초)으로 호출 부하를 줄입니다.
  • 중복 제거·정규화: 동일 id는 전역에서 한 번만 유지합니다. 원시 항목은 parse_item으로 요약·대분야 코드 매핑·세부분야·닉네임·좋아요·태그·등록일 등으로 정규화한 뒤, 등록일 기준 내림차순으로 다시 번호(index)를 매깁니다.
  • 실패 처리: 네트워크 오류 시 지수 백오프로 최대 3회 재시도합니다. 치명적 오류는 CrawlError로 상위(FastAPI /api/crawl)에 전달됩니다.

3. Raw data 분석 방법

  • 입력: 정규화된 ideas 배열과 선택적 crawled_at을 받습니다. 구현은 modoo_analytics.compute_analytics입니다.
  • 집계 축: 대분야·세부분야·도전자(닉네임)·좋아요·등록일 등 필드 기반으로 건수·비율·분포(히스토그램)·참여자 수·시계열 등을 계산합니다.
  • 텍스트(요약) 처리: 한글 연속 글자(가-힣, 2글자 이상)를 토큰으로 잡고, 사전 정의된 불용어 집합을 제외한 뒤 빈도 상위 키워드를 산출합니다.
  • 프런트 표시: /analytics는 /api/analytics 또는 배포 시 /data/analytics.json의 결과를 Chart.js 등으로 시각화합니다. 수집 직후 export_for_research 경로에서는 동일 로직으로 research/data/analytics.json이 갱신됩니다.

4. 인사이트(Insight) 분석 방법

  • 입력·역할: 동일한 ideas 전수에 대해 modoo_insight.compute_insight가 정량·정성·전략 블록을 한 번에 계산합니다. 외부 LLM을 호출하지 않고, 규칙·키워드·휴리스틱 기반입니다.
  • 정량: 분야별 건수, 문제·가치 유형(키워드 규칙 목록과 요약 텍스트 매칭), 고객군·B2B/B2C/B2G 추정, AI 관련 언급 비율, 요약 토큰 유사도로 만든 그래프의 연결 요소(클러스터) 등을 집계합니다.
  • 정성: 창업 동기 유형, "과장형" vs "실현형" 키워드, 사회·임팩트·로컬 상위 키워드, 기술 언급은 많으나 시장·고객 언급이 적은 사례 표 등을 텍스트 패턴으로 뽑습니다.
  • 전략 문장: 위 지표를 조합해 짧은 해석 문장(카피)을 생성하는 레이어로, 보고서형 UI(/insight)에 바인딩됩니다.
  • 데이터보내기: 수집 완료 시 research/data/insight.json과 출처 메타 insight_data_source.json이 함께 기록되어, 화면과 파일이 동일 전수를 가리키는지 검증할 수 있습니다.
